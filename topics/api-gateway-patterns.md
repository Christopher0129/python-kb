# API 网关模式专题

这个专题关注网关层如何承接鉴权、路由、限流、聚合和协议转换等横切能力。

## 适用场景

- 对外开放 API 或多端接入
- 需要统一鉴权、限流和审计
- 多后端服务需要统一出口

## 常见主题

- 路由与聚合
- 鉴权透传
- 限流与配额
- 请求改写与头信息治理
- 网关与应用职责边界

## 设计要点

- 网关适合承接共性横切逻辑，不适合堆积领域规则
- 应用内鉴权和网关鉴权要避免互相矛盾
- 头透传、trace id、租户信息要统一约定
- 网关策略需要版本化和回滚能力
