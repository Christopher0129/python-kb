# 审计日志专题

这个专题关注哪些操作需要留痕、怎么记录以及如何让审计日志可追踪、可查询、可合规。

## 适用场景

- 管理后台、高风险接口、权限变更
- 配置修改、发布操作、数据导出
- 合规要求需要追踪“谁在什么时候做了什么”

## 常见主题

- 操作主体与租户信息
- 操作对象与前后值
- 请求来源与 trace id
- 防篡改与保留周期
- 查询与告警

## 设计要点

- 审计日志和普通业务日志的目标不同，字段模型要分开
- 高风险操作应记录结果、原因和审批信息
- 不能只记录“发生了”，还要能还原上下文
- 脱敏和最小必要原则要贯穿审计设计
