# 事故响应专题

这个专题关注线上故障发生后，团队如何定位、缓解、恢复并沉淀改进。

## 适用场景

- API 5xx 激增
- 任务积压或消息堆积
- 数据漂移、模型效果突降
- 发布后核心链路退化

## 常见主题

- 告警分级
- 值班与升级路径
- 缓解动作
- 回滚与降级
- 事后复盘

## 设计要点

- 响应流程要明确谁负责判断、谁负责执行、谁负责沟通
- 预先准备限流、熔断、回滚、切流、关开关等操作手段
- 事故处理中的时间线、证据和决策应留痕
- 复盘要追到系统性改进，而不止停留在“某人操作失误”
