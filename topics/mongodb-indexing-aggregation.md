# MongoDB 索引与聚合专题

这个专题关注 MongoDB 的查询性能、聚合管道和索引命中，而不是把聚合当作“能跑就行”的黑盒。

## 适用场景

- 聚合管道越来越长，线上耗时不稳定
- 排序、过滤和数组字段查询难以同时兼顾
- 文档条数增大后，索引与内存压力明显上升

## 常见主题

- 单字段、复合、多键索引
- 聚合管道阶段顺序
- 覆盖查询与投影
- 排序与内存使用
- `explain()` 与慢查询分析

## 设计要点

- 聚合优化的第一步通常不是加机器，而是减少无效阶段和过早展开的数据量。
- 索引顺序应和最稳定的过滤、排序路径匹配，而不是按字段自然顺序堆叠。
- 数组字段、文本字段和多维过滤常常需要单独设计索引策略。
- 聚合逻辑一旦承载核心报表能力，就应纳入容量规划和回归基线。
