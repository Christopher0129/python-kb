# MySQL 锁冲突与争用专题

这个专题关注 MySQL 在高并发写入下的锁等待、热点更新、死锁和事务阻塞问题。

## 适用场景

- 更新同一批热点记录时吞吐明显下降
- 事务偶发超时或死锁
- 排查时发现大量线程卡在 lock wait

## 常见主题

- 行锁、间隙锁、next-key lock
- 热点行与热点索引
- 死锁检测与回滚
- 长事务阻塞
- 批量更新与分批提交

## 设计要点

- MySQL 争用问题本质上常常不是“数据库不够快”，而是访问路径过于集中。
- 范围更新、非唯一索引扫描和不稳定的访问顺序，都会放大锁范围。
- 长事务会把本来可恢复的小抖动放大成整条链路阻塞。
- 争用治理通常要同时改 SQL、事务粒度、批次大小和业务重试策略。
