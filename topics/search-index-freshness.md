# 搜索索引新鲜度专题

这个专题关注内容更新到可被搜到之间的延迟、丢失和一致性问题。

## 适用场景

- 文档刚更新后很久搜不到
- 搜索结果与主数据库状态不一致
- 增量索引链路复杂，定位延迟来源困难

## 常见主题

- 增量更新
- 同步延迟
- 索引积压
- 回放补数
- 一致性窗口

## 设计要点

- 索引新鲜度要有明确的 SLO，而不是笼统地说“最终一致”。
- 排障时要区分数据源延迟、消息积压、索引写入失败和 alias 未切换等问题。
- 高价值内容变更链路通常需要优先级和快速通道。
- 指标应覆盖更新时间差、积压深度、失败重试和补数成功率。
