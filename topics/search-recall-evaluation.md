# 搜索召回评测专题

这个专题关注搜索或 RAG 系统在排序前的召回层是否真的把“该找回来的候选”拿回来了。

## 适用场景

- 排序调得很多，但用户仍然搜不到关键结果
- 召回策略越来越多，不清楚哪层在丢失候选
- 需要离线评估关键词召回、向量召回和混合召回

## 常见主题

- 标注集
- top-k 命中
- 召回覆盖率
- 离线对比
- 失败样例

## 设计要点

- 召回评测和排序评测要拆开做，否则很难定位问题层次。
- 标注集要覆盖头部查询、长尾查询和真实失败样例。
- 召回层关注“有没有找回来”，不是“排在第几位”。
- 离线评测结果要和线上日志对齐，否则容易出现实验集乐观偏差。
