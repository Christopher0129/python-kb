# 测试金字塔与切片专题

这个专题关注测试数量和粒度怎么分布，避免所有验证都堆到慢而脆的端到端测试上。

## 适用场景

- 项目测试越来越慢
- 修一个小功能却要跑整套集成链路
- 团队对“单元 / 集成 / 接口 / E2E”边界认知不一致

## 常见主题

- 测试金字塔与测试蜂窝图
- 单元测试与集成测试的边界
- slice test 与全链路测试
- fixture 共享范围
- 外部依赖替身策略

## 设计要点

- 绝大多数业务规则应先落在快而稳定的单元或 slice test
- 端到端测试数量应少但关键路径明确
- 测试切片按架构边界设计，例如 API 层、仓储层、事件处理层
- 每一层都要回答“失败时能否快速定位”
